Data Cluster Setup (CASS 3/UBUNTU 16.04 LTS)

  • Update OS
  • Install OracleJDK as follows:
  • Install Cassandra
  • Tuning the OS
  • Set up a multi-node cluster
  • Start the cluster
  • Debugging

Update OS

apt-get update
apt-get upgrade
apt-get -y install emacs23-nox (or install whatever editor you want to use)

 

Install OracleJDK as follows:

(Base on these (https://www.digitalocean.com/community/tutorials/how-to-install-java-on-ubuntu-with-apt-get) instructions.)

apt-get -y install python-software-properties
add-apt-repository -y ppa:webupd8team/java
apt-get update
apt-get -y install oracle-java8-installer


Verify Java version 1.8:

java -version

 

INSTALL Python-support  (no longer included in Ubuntu 16.04 and required by Cassandra)

wget http://launchpadlibrarian.net/109052632/python-support_1.0.15_all.deb
sudo dpkg -i python-support_1.0.15_all.deb

 

Install Cassandra

Based on [these](https://docs.datastax.com/en/cassandra/3.0/cassandra/install/installDeb.html) instructions.

Add repo:

echo "deb http://debian.datastax.com/community stable main" | tee -a /etc/apt/sources.list.d/cassandra.sources.list

Add the DataStax repository key to your aptitude trusted keys:

curl -L https://debian.datastax.com/debian/repo_key | apt-key add -

Install Cassandra:

We are using Spring Data Cassandra 1.5 (http://docs.spring.io/spring-data/cassandra/docs/1.5.1.RELEASE/reference/html/#new-features.1-5-0), so we should install the latest Cassandra 3.0.x (as DataStax Enterprise 5.0 uses this version so it seems like that is where the active development will be).

apt-get remove cassandra [if previously installed]
apt-get install dsc30
apt-get install cassandra-tools

Stop the server and clear the data:

service cassandra stop ; rm -rf /var/lib/cassandra/data/system/*


Tuning the OS

Tune limits:
(as per [here](http://docs.datastax.com/en/cassandra/2.2/cassandra/install/installRecommendSettings.html))

Ensure that the following settings are included in the /etc/security/limits.d/cassandra.conf file:

<cassandra_user> - memlock unlimited
<cassandra_user> - nofile 100000
<cassandra_user> - nproc 32768
<cassandra_user> - as unlimited


For all installations, add the following line to /etc/sysctl.conf (sets the max memmap areas):

vm.max_map_count = 131072


To make the changes take effect, reboot the server or run the following command:

sysctl -p


Turn off swap for the nonce:

swapoff --all


To make this change permanent, remove all swap file entries from /etc/fstab.

To check swap:

grep -i swap /proc/meminfo


Tune JVM (if needed) as per [here](http://docs.datastax.com/en/cassandra/2.0/cassandra/operations/ops_tune_jvm_c.html)

Set up a multi-node cluster

(based on [these](http://docs.datastax.com/en/cassandra/2.2/cassandra/initialize/initSingleDS.html) instructions; IP addresses will of course differ)

Head node internal IP address: 172.16.100.7 (root@128.194.6.148)

Cluster node IP addresses:

- node0: 172.16.100.1 (root@128.194.6.152)
- node1: 172.16.100.2 (root@128.194.6.149)
- node2: 172.16.100.3 (root@128.194.6.157)
- node3: 172.16.100.4 (root@128.194.6.153)
- node4: 172.16.100.5 (root@128.194.6.151)
- node5: 172.16.100.6 (root@128.194.6.150)

Stop Cassandra and clear data:

service cassandra stop
rm -rf /var/lib/cassandra/*


(also do this if a node won't start up, for examples because:

- org.apache.cassandra.exceptions.ConfigurationException: Cannot add table 'role_permissions' to no
n existing keyspace 'system_auth'.

- org.apache.cassandra.exceptions.ConfigurationException: Found system keyspace files, but they couldn't be loaded!

)

Backup previous cassandra.yaml:

cp /etc/cassandra/cassandra.yaml /etc/cassandra/cassandra.yaml.orig

Set the following properties in /etc/cassandra/cassandra.yaml to the following values for each node (keep all other settings as is; don't start from an empty file):

```
cluster_name: 'CGISensorThings'
num_tokens: 256
seed_provider:
- class_name: org.apache.cassandra.locator.SimpleSeedProvider
parameters:
- seeds: "172.16.100.1,172.16.100.4"
listen_address: 172.16.100.?
rpc_address: 172.16.100.?
endpoint_snitch: SimpleSnitch
```


Start the cluster

Start each seed node one-at-a-time:

service cassandra start


Check on the status (status should be UN; Up Normal):

nodetool status


Then start the rest of the nodes (may have to wait up to

two minutes between each node):

service cassandra start

 

Check on the status (status should be UN; Up Normal):

nodetool status

Debugging

If you are running into issues, you can find the cassandra logs here- /var/logs/cassandra

Use system.log to view start up errors. For example:

  • Nodetool status renders:

    • nodetool: Failed to connect to '127.0.0.1:7199' - ConnectException: 'Connection refused (Connection refused)'.

  • View the logs to see start up errors.
