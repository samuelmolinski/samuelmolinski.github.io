Raspberry Pi Configuration

Original CGI Document

  1. Setup Network Connectivity:

    Command
    sudo nano /etc/dhcpcd.conf
    Configuration File
    # Note: Under "nohook lookup-hostname" text
 
interface eth0

static ip_address=<ip>/16
static routers=<ip_gateway>
static domain_name_servers=<ip_dns>
    ctrl + x to exit
Y (Hit this letter and then hit enter key)

     

    Command
    sudo nano /etc/network/interfaces
    Configuration File
    # interfaces(5) file used by ifup(8) and ifdown(8)

# Please note that this file is written to be used with dhcpcd
# For static IP, consult /etc/dhcpcd.conf and 'man dhcpcd.conf'

# Include files from /etc/network/interfaces.d:
source-directory /etc/network/interfaces.d

auto lo
iface lo inet loopback

iface eth0 inet static
  address <ip address>
  netmask <netmask>
  gateway <gateway>
  dns-nameservers <dns>

allow-hotplug wlan0
iface wlan0 inet manual
    wpa-conf /etc/wpa_supplicant/wpa_supplicant.conf

allow-hotplug wlan1
iface wlan1 inet manual
    wpa-conf /etc/wpa_supplicant/wpa_supplicant.conf
    ctrl + x to exit
Y (Hit this letter and then hit enter key)

     

    Command
    sudo nano /etc/modprobe.d/raspi-blacklist.conf

    Add the following to disable WiFi and Bluetooth on the Raspberry Pi:

    Configuration File
    #wifi
blacklist brcmfmac
blacklist brcmutil
#bt
blacklist btbcm
blacklist hci_uar
    ctrl + x to exit
Y (Hit this letter and then hit enter key)

     

    Command
    sudo reboot

  2. Update and upgrade your local repository in the Raspian Jessie Lite OS:

    Commands
    sudo apt-get update
sudo apt-get upgrade

  3. Install software needed for running sensor / transmit code:

    Commands
    sudo apt-get install python3
sudo apt-get install python3-rpi.gpio
sudo apt-get install python3-wheel
sudo apt-get install python3-pip
sudo easy_install-3 -U pip
 
sudo wget https://pypi.python.org/packages/e4/b0/286e8a936158e5cc5791d5fa3bc4b1d5a7e1ff4e5b3f3766b63d8e97708a/requests-2.14.2-py2.py3-none-any.whl#md5=a1b2f02e7ba45dff7c76f621cde042c4
 
sudo wget https://pypi.python.org/packages/dd/85/519354e995d8a926ce3121034dc2144a5ae70435dad3e1155a19bbde8011/requests_toolbelt-0.7.1-py2.py3-none-any.whl#md5=978e1d4dad99ebdb780bd95f6473e98a
 
sudo pip3 install requests-2.13.0-py2.py3-none-any.whl 
sudo pip3 install requests_toolbelt-0.7.1-py2.py3-none-any.whl

  4. Install Git and clone the LEaRN repository to install the program:

    Commands
    sudo apt-get install git
 
git clone <learn repo>
 
cd learn/sensors/
 
sudo python3 setup.py install
 
cd
 
sudo rm -rf <learn repo>

  5. Setup the sensor shell script in home directory:

    Commands
    which sensor_raspi_sample  # Shows the absolute path of the sampler script

nano sensor.sh

    Add parameters in sensor.sh file:  (NOTE:  Feature of Interest ID and Datastream ID will be provisioned with unique IDs)

    Configuration File
    #!/bin/sh
export CGIST_FOI_ID="1111111-1111-1111-1111-111111111111"
export CGIST_DS_ID_MQ131="222222-2222-2222-2222-222222222222"
/usr/local/bin/sensor_raspi_sample > /var/log/sensor.log 2>&1
    ctrl + x to exit
Y (Hit this letter and then hit enter key)

    Set script as an executable and give it the following permissions:

    Commands
    chmod +x sensor.sh 
chmod 755 sensor.sh

  6. Setup the transmit shell script in home directory:

    Commands
    which sensor_transmit   # Shows the absolute path of the transmitter script
 
nano transmit.sh

    Add parameters in transmit.sh file:  (NOTE: ID fields will be provisioned with unique IDs)

    Configuration File
    #!/bin/sh
export CGIST_URL="https://api.learnlafayette.com/device/api/v1.0/CreateObservations"
export CGIST_AUTH_URL="https://api.learnlafayette.com/device/api/auth/login"
export JWT_ID="1111111-1111-1111-1111-111111111111"
export JWT_KEY="222222-2222-2222-2222-222222222222"
/usr/local/bin/sensor_transmit > /var/log/sensor.log 2>&1
    ctrl + x to exit
Y (Hit this letter and then hit enter key)

    Set script as an executable and give it the following permissions:

    Commands
    chmod +x transmit.sh 
chmod 755 transmit.sh

  7. Setup Sensor and Transmit services:

    Command
     sudo nano /lib/systemd/system/sensor.service
    Configuration File
    [Unit]
Description=Sensor Spool Service
After=multi-user.target

[Service]
Type=idle
ExecStart=/home/pi/sensor.sh

[Install]
WantedBy=multi-user.target
    ctrl + x to exit
Y (Hit this letter and then hit enter key)

     

    Command
     sudo nano /lib/systemd/system/transmit.service
    Configuration File
    [Unit]
Description=Transmit Sersor Data Service
After=multi-user.target

[Service]
Type=idle
ExecStart=/home/pi/transmit.sh

[Install]
WantedBy=multi-user.target
    ctrl + x to exit
Y (Hit this letter and then hit enter key)


    Set permissions and configure systemd for the new services:

    Commands
     sudo chmod 644 /lib/systemd/system/sensor.service 
 sudo chmod 644 /lib/systemd/system/transmit.service 
 sudo systemctl daemon-enable
 sudo systemctl daemon-reload
 sudo systemctl enable sensor.service
 sudo systemctl enable transmit.service


    Setup Cron job to restart transmitter service every 15 minutes:  

    Commands
     crontab -e
    Configuration File
    # Edit this file to introduce tasks to be run by cron.
#
# Each task to run has to be defined through a single line
# indicating with different fields when the task will be run
# and what command to run for the task
#
# To define the time you can provide concrete values for
# minute (m), hour (h), day of month (dom), month (mon),
# and day of week (dow) or use '*' in these fields (for 'any').#
# Notice that tasks will be started based on the cron's system
# daemon's notion of time and timezones.
#
# Output of the crontab jobs (including errors) is sent through
# email to the user the crontab file belongs to (unless redirected).
#
# For example, you can run a backup of all your user accounts
# at 5 a.m every week with:
# 0 5 * * 1 tar -zcf /var/backups/home.tgz /home/
#
# For more information see the manual pages of crontab(5) and cron(8)
#
# m h  dom mon dow   command

*/1 * * * * sudo -u pi scp /var/log/sensor.log <Where log file gets copied to every minute>

*/15 * * * * sudo systemctl restart transmit.service
    ctrl + x to exit
Y (Hit this letter and then hit enter key)

     

  8. Setup SQLite in RAM:

    Command
     sudo nano /etc/fstab

    Add this line:

    Configuration File
    tmpfs /var/spool/mqueue tmpfs defaults,noatime,nosuid,mode=0700,gid=12,size=30m 0 0
    ctrl + x to exit
Y (Hit this letter and then hit enter key)

     

  9. Set final permissions before rebooting Raspberry Pi:

    Commands
    sudo touch /var/spool/mqueue  # This is if the file doesn't exist.
sudo chmod -R 777 /var/spool/mqueue
sudo chmod -R 777 /var/log/
    Command
    sudo reboot

  10. Check the status of the new services running after rebooting:

    Commands
    sudo systemctl status sensor.service
 
sudo systemctl status transmit.service

  11. Check the log to see the application at work:

    Command
     tail -f /var/log/sensor.log
